## Requirements
### Requirement: API-Driven Dashboard
The dashboard SHALL retrieve all data it displays by calling the FastAPI backend's endpoints, and SHALL NOT access the dataset, model files, MLflow, or Prefect directly.

#### Scenario: Dashboard calls API only
- **WHEN** the dashboard renders any view (EDA, prediction, app details)
- **THEN** every piece of displayed data was retrieved via an HTTP call to the FastAPI service

### Requirement: EDA Visualization Display
The dashboard SHALL display the EDA visualizations and summary statistics produced by the data pipeline.

#### Scenario: EDA view
- **WHEN** a user opens the EDA section of the dashboard
- **THEN** they see summary statistics, the correlation view, and the univariate/bivariate charts generated by the data pipeline

### Requirement: Prediction Interface
The dashboard SHALL let a user enter house feature values and view the predicted sale price.

#### Scenario: User requests a prediction
- **WHEN** a user fills in the house feature form and submits it
- **THEN** the dashboard calls the prediction API and displays the returned predicted SalePrice

### Requirement: Application Details Display
The dashboard SHALL display the built-in application/deployment details exposed by the API (sub-objective 3).

#### Scenario: App details view
- **WHEN** a user opens the application-details section of the dashboard
- **THEN** they see at least four details (model registry info, experiment info, pipeline/deployment info, recent run info) retrieved from the API
